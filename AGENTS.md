## 프로젝트 루트

- `/home/runner/workspace`가 곧 프로젝트 루트다.
- `site-ai-score/` 같은 중첩 프로젝트 폴더를 만들지 않는다.
- 모든 명령은 `/home/runner/workspace`에서 실행한다.
- `client`, `server`, `shared`, `prisma`는 루트 바로 아래의 정상 소스 폴더다.

## 협업 방식

1. 사용자는 비개발자이며 Replit Shell 명령을 복사해 실행한다.
2. 한 이슈 단위로 명령을 제공한다.
3. 실행 결과가 회신되지 않으면 실행되지 않은 것으로 본다.
4. 실제 파일을 조회한 뒤 수정한다.
5. 기존 기능을 삭제하거나 비활성화하지 않는다.
6. 긴 수정은 업로드형 단일 `.cjs` 패치로 진행한다.
7. 패치는 대상 존재, 기존 코드 일치, 중복 적용 여부를 먼저 검사한다.
8. 코드 수정 후 `check → test → build → git add/commit/push → git status -sb` 순서로 확인한다.
9. 테스트 위치가 개발모드인지 배포 사이트인지 명시한다.
10. 모든 날짜·시간 계산과 표시는 KST를 기준으로 한다.

## 제품 원칙

- 실제 공개 URL에서 관찰되는 결과를 기준으로 검사한다.
- 점수와 pass/fail은 규칙 기반으로 계산한다.
- 모든 감점과 실패에는 URL, 검사값, 증거를 남긴다.
- 검색용 봇과 학습용 봇을 구분한다.
- 수정 전후는 같은 규칙 버전과 같은 조건으로 비교한다.
- 고객 사이트를 직접 수정하거나 배포하지 않는다.

전체 요구사항은 `SITEAISCORE_FINAL_DEVELOPMENT_PROMPT.md`를 따른다.
