# Site AI Score 정밀진단·AI 답변 성과 설계

- 설계 버전: 2026.06-ai-answer-v1
- 기준 시간대: KST
- 상태: 기반 데이터 모델과 규칙 계산부터 구현

## 1. 목적

정밀진단은 기존 QUICK 기술 점수를 대체하지 않는다.

- QUICK 기술 진단: 공개 URL의 접근성, 초기 HTML, 구조, 링크, JSON-LD 등
- AI 답변 성과: 실제 웹 검색 기반 AI 답변에서 브랜드·사이트가 발견되고 정확히 설명되는 정도

두 결과는 서로 다른 의미이므로 화면과 PDF에서 별도로 표시한다.

## 2. 공식 표시 명칭

소비자용 ChatGPT 화면의 순위나 추천 결과를 복제한다고 표현하지 않는다.

서비스 화면에는 다음처럼 표시한다.

- OpenAI 웹 답변 테스트
- 실행 모델
- 실행 시각(KST)
- 질문 세트와 반복 횟수
- 검색 출처와 대상 사이트 인용 여부
- 테스트 환경에 따라 결과가 달라질 수 있다는 안내

## 3. 질문 유형

- BRAND: 브랜드·서비스 직접 질문
- DISCOVERY: 브랜드명을 넣지 않은 탐색 질문
- FEATURE: 기능·지원 범위 질문
- USE_CASE: 이용 대상·활용 사례 질문
- TRUST: 요금·운영 주체·개인정보·문의 등 신뢰 질문
- COMPARISON: 대안·비교·추천 문맥 질문
- CUSTOM: 운영자 직접 질문

질문은 삭제 대신 ACTIVE·ARCHIVED 상태로 관리한다. 검사 실행 시 질문 문구와 기대 사실을 AiAnswerRun에 스냅샷으로 저장한다.

## 4. 기준정보

기존 SiteFact를 정답 기준정보로 사용한다.

예시 키:

- service_definition
- primary_features
- target_audience
- usage_steps
- supported_platforms
- pricing
- data_handling
- operator
- contact
- unsupported_capabilities

각 질문은 expectedFactKeys로 필요한 기준정보 키를 연결한다. 기준정보가 없으면 사실 정확도를 임의 판정하지 않고 미측정으로 남긴다.

## 5. 실제 실행 기록

각 질문과 반복 실행마다 다음을 저장한다.

- provider와 model
- 질문 코드·종류·문구
- 실행 번호
- 응답 ID
- 답변 전문과 SHA-256
- 브랜드 언급 여부
- 대상 도메인 인용 여부
- 본문 인용 목록
- 검색에 사용된 전체 출처 목록
- 자동 측정값
- 기준정보 비교 결과
- 토큰 사용량
- 오류 코드·메시지
- 시작·완료 시각

원문 질문과 기대 사실은 실행 당시 스냅샷을 보존한다.

## 6. 성과 지표

- 답변 완료율
- 브랜드 언급률
- 대상 사이트 인용률
- 사실 정확도
- 답변 완전성
- 반복 답변 일관성
- AI 답변 성과점수

QUICK 점수와 AI 답변 성과점수는 합산하지 않는다.

### AI 답변 성과점수 배점

- 대상 사이트 인용률: 30
- 브랜드 언급률: 15
- 사실 정확도: 30
- 답변 완전성: 15
- 반복 일관성: 10

사실 정확도와 완전성이 평가되지 않았으면 종합 성과점수를 생성하지 않는다. 반복이 한 번뿐이면 일관성 10점은 제외하고 나머지 측정값으로 재정규화하며 scoreCoverage를 함께 표시한다.

## 7. 자동 판정과 모델 평가

자동으로 확정 가능한 항목:

- 답변 생성 성공 여부
- 브랜드 문자열 언급 여부
- 대상 도메인 URL 인용 여부
- 인용·출처 URL
- 반복 실행 결과 수

모델 평가가 필요한 항목:

- 기준정보와의 사실 일치
- 질문에 대한 답변 완전성
- 근거 없는 주장
- 상충하거나 잘못된 정보

모델 평가 결과는 근거 사실별 판정과 이유를 JSON으로 저장한다. 모델이 임의로 QUICK 점수나 규칙 PASS·FAIL을 바꾸지 못한다.

## 8. 실행 단계

1. 사이트 기준정보 입력·수정
2. 기본 질문 생성
3. 운영자 질문 추가·수정·보관
4. DEEP 검사 생성
5. 기존 HTTP·초기 HTML·렌더링 검사
6. 질문별 OpenAI 웹 답변 테스트
7. 답변·인용·전체 출처 저장
8. 기준정보 기반 사실 평가
9. 반복 일관성과 성과지표 계산
10. 결과 화면과 PDF 표시

## 9. 비용·안전 원칙

- API Key는 Replit Secrets의 서버 환경에만 둔다.
- 질문 수와 반복 횟수에 상한을 둔다.
- 한 검사 안에서는 순차 실행을 기본으로 한다.
- 무한 재시도를 금지한다.
- 외부 API 실패는 DEEP 검사를 PARTIAL로 남기고 기존 기술 진단을 삭제하지 않는다.
- 숨겨진 추론 과정은 저장하지 않는다.
- 응답·오류 저장 전 토큰·쿠키·인증정보를 정제한다.
- 소비자용 ChatGPT 검색 순위 또는 향후 추천을 보장하지 않는다.

## 10. 후속 구현 순서

1. 이 문서의 DB 모델·성과 계산 기반
2. 기준정보와 질문 CRUD API
3. 모바일 우선 질문·기준정보 관리 화면
4. OpenAI Responses API web_search 실행기
5. 인용·전체 출처 파서
6. 기준정보 평가기
7. DEEP 워커 연결
8. 정밀 결과 화면과 PDF
9. 동일 질문 반복 실행과 전후 비교


## 11. OpenAI 웹 답변 실행 연결

- Responses API의 hosted `web_search` 도구 사용
- `tool_choice: "required"`로 질문마다 웹 검색 실행 강제
- `web_search_call.action.sources` 전체 검색 출처 저장
- 답변의 `url_citation` 인용 저장
- 기본 검색·평가 모델: `gpt-5.4-mini`
- 검색 답변과 기준정보 사실 평가는 별도 요청으로 분리
- 기본 반복 횟수: 질문당 2회
- API 키 미설정 시 정밀진단 시작 차단
- 기술 진단 성공 후 일부 AI 요청이 실패하면 전체 검사를 PARTIAL로 보존
- 실제 답변, 인용, 전체 출처, 자동 측정값, 사실 평가, 토큰 사용량 저장
- 소비자용 ChatGPT 검색 순위를 재현하거나 향후 노출을 보장하지 않음
